# Javascript 변천사

> 이 문서는 [김민태의 프론트엔드 아카데미](https://fastcampus.co.kr/category_academy_kmt) 강의를 바탕으로 작성되었습니다.

## Javascript의 탄생

- 1995년에 넷스케이프 브라우저에 처음 탑재되어 나왔다.
  - 이때 이름은 LiveScript 였다.
  - 작은 브라우저에서 동작하면서 HTML을 간단하게 조작하기 위해 만들어진 언어라서, 배울게 별로 없었다.
  - 당시에 존재하던 Explorer에도 JScript라는 스크립트 언어가 탑재됐었다.
  - LiveScript와 JScript는 비슷하지만 달랐다.

## 표준화 작업

- `ECMA`라는 표준 단체에서 Javascript라는 이름으로 표준을 계속 개발한다.
  - 그래서 Javascript의 공식 명칭은 EcmaScript이다.
- 2009년에 나온 버전 5.0, EcmaScript 5.0까지는 아주 크게 발전하지는 못했다.
  - 이유는 어도비의 Flash Player가 전성기였기 때문, HTML과 Javascript는 인기가 없었다.
  - Flash Player안에서 채택된 ActionScript를 JavaScript표준으로 만드려고 Adobe가 노력도 했다.

## 아이폰의 등장

- 애플의 아이폰이 나왔을 때, Flash Player, Flash가 표준기술이 아니었기 때문에 문제가 많았다.
  - 스티브 잡스가 `No Flash 선언`
  - Flash Player는 점점 사용하지 않게 된다.

## 모던 자바스크립트

- EcmaScript 5.0과 ECMAScript2015 사이에는 아무 버전도 있지 않았다.
- ES2015버전부터 `모던 자바스크립트`라고 이야기를 한다.
- 그럼에도 불구하고 `5.0버전`이 주도적인 버전이다.
- ES2015이후 버전 자바스크립트를 지원하지 않는 브라우저들도 존재한다.
  - 개발자들은 최신버전으로 개발하지만,`5.0 버전으로 변환하는 변환기(트랜스 파일러)`를 사용한다.
