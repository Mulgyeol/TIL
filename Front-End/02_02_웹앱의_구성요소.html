# 웹앱의 구성요소

  • HTML
  • CSS
  • Javascript

# 실행 관점의 구성요소

# 웹앱이 실행된다?

  • 사용자가 브라우저를 실행한 다음, 주소창에 특정 홈페이지 주소를 입력한다.
  • 웹 사이트에 접속하면 HTML, CSS, Javascript 파일들이 물리적으로 브라우저에 전송한다.
  • 브라우저가 파일을 해석해서 실행시키면서 웹 앱이 동작한다.

# 각각 파일의 역할

  • HTML : UI를 기본적으로 만든다.
  • CSS : UI의 비주얼적인 요소들을 담당한다.
  • Javascript : 동적인, 프로그래밍적 요소를 담당한다.

# 단순히 웹앱의 구성요소와 달라진 점?

  • 웹 앱을 실행시키는 역할을 하는 브라우저라는 것을 고려한다.
  • 이 웹 앱이 실행되는 시간, 런타임이라고 한다.
  • 브라우저는 런타임 환경을 제공하는 환경이라고 보통 이야기 한다.

# Node.js

  • 예전에는 브라우저만이 Javascript를 실행시키는 런타임 환경을 제공하는 유일한 환경이었다.
  • Node.js 라는 어플리케이션이 나오면서 브라우저가 아니어도 Javascript를 실행시킬 수 있는 환경이 다수 만들어졌다.

# CSR && SSR

# 개요

  • UI를 만드는 것이 HTML이고, Javscript를 이용해서 HTML을 조작할 수 있기 때문에, Javascript를 이용해서도 UI를 제공할 수가 있다.
  • 결론적으로는 어떤 식으로든 UI를 만들기 위해서는 최종적으로 HTML 형태가 만들어져 있어야 한다.
  • 따라서, HTML을 어디에서 만드는지가 중요하다.

# CSR(Client Side Rendering)

  • 브라우저에서 실행된 HTML이 Javscript에 의해서 조작되는 방식
  • 최초 전송된 HTML에는 내용이 거의 들어있지 않다.
  • Javascript가 실행되면서 필요한 UI를 그때끄때 만들어낸다.

# SSR(Server Side Rendering)

  • 웹 서버에서 주도적으로 HTML을 만들어 브라우저로 전송하는 방식

# CSR과 SSR 중 뭐가 더 좋은가?

  • 앱의 성격에 따라서 효율적인 방식을 사용해야한다.
  • 앱의 성격에 맞지 않는 렌더링 방식을 선택했을 때는 비효율이 발생한다.

# 그 외 구성요소

# 그래픽 시스템

  • 웹앱 자체는 기본적으로 그래픽 시스템을 HTMl과 CSS로 표현할 수 있다.
  • 그렇지만, 차트를 만든다거나 애니메이션을 표현한다거나 혹은 3D를 표현한다거나 하는 더 적극적인 그래픽 시스템이 필요한 경우가 있다.
    • 이 경우, HTML과 CSS로는 한계가 있다.
  • 그래서 Javascript로 3D그래픽, 2D 그래픽을 표현할 수 있게끔 제공해주는 캔버스라는 태그가 있다.
    • 캔버스는 3D, 2D 그래픽을 표현하기 위한 영역만 제공해주고, 실직적 그래픽 작업은 Javascript로 이루어진다.
