# 빠르게 배워야할 것, 나중에 배워야할 것, 반복해 배워야 하는 것

> 이 문서는 [김민태의 프론트엔드 아카데미](https://fastcampus.co.kr/category_academy_kmt) 강의를 바탕으로 작성되었습니다.

## 세 가지 유형의 기술

- 변하지 않는 기술
  - 네트워크
  - 운영체제
  - 컴퓨터 시스템
  - 논리학
  - 대수학
- 느리게 변하는 기술
  - 프로그래밍 언어
  - 프로그래밍 패러다임
  - 자료구조
  - 알고리즘
  - 보안
- 빠르게 변하는 기술
  - 프레임워크
  - 라이브러리
  - UI/UX
  - 디자인패턴

변하지 않는 기술일수록 학습비용이 높고,
빠르게 변하는 기술일수록 학습비용이 낮다.

## 학습 전략 수립

- 쉬운 것 부터 배우는 것이 효과적이다.
- 그러나, 빠르게 변하기 때문에 새로운 기술을 계속 배워야한다.
- 상대적으로 느린 변화 속도의 기술을 익히면 그것보다 빠른 변화 주기를 가진 지식들을 학습하는데 훨씬 도움을 준다.

## 프로그래밍 역량에 대한 생각

- `기술을 안다`, `Javascript를 배워서 할 줄 안다.` `Typescript를 배워서 할 줄 안다, 앱을 만들어봤다.` 라고 이야기 할 수도 있다. 그러나,
- 구체적으로 내가 어떤 이미지를 갖고 `내가 이런 역량을 향상시켜야지`, `이런 부분이 부족하니까 이런 기술을 공부해야지` 처럼,
- 이미지를 갖고 하는 것과 이미지 없이 학습하는 것의 효과는 큰 차이가 난다.

## 프로그래밍 4가지 역량

1. **일관성**
2. **유연성**
3. **확장성**
4. **독립성**

학습할 때, 위 프로그래밍 역량에 맞추어 전략을 세워나가면 더욱 효과적으로 학습할 수 있다.

## 빠르게 배워야할 것, 나중에 배워야할 것, 반복해 배워야 하는 것

- 빠르게 배워야할 것 : 변화에 빠른 트렌디한 기술들, 적은 학습비용으로 빠르게 습득
- 나중에 배워야할 것 : 상대적으로 시간 투자를 많이해야하고 배우는데 학습비용이 많이 들고 어려운 내용 -> **꾸준히 학습**
- 반복해 배워야할 것 : 변화의 속도가 느린 지식들. 어렵고 허들이 높은 지식들
