# 모던 자바스크립트와 개발 환경

이 문서는 김민태의 프론트엔드 아카데미 (opens new window) 강의를 바탕으로 작성되었습니다.

ES2015 버전부터 모던 Javascript라고 보통 이야기한다.

# 급변한 프론트엔드 개발 환경

  • 핵심적인 이유 : 웹앱의 규모가 커지고 복잡해졌다.

# 모듈

  • ES2015 버전부터 모듈 스펙을 지원
  • 모듈 : 파일과 파일 간에 특정 파일의 어떤 기능을 사용하기 위해서 불러들여서 쓸 수 있는 기능
    • 사용 방법 : import, export 문법
      • html에서 script 태그를 이용해서 자바스크립트 파일을 불러오는 것과는 다르다.
      • Javascript 안에서 다른 Javascript 파일을 불러오는 것을 의미한다.
      • 파일이 많아 질수록 효율성이 증가한다.
    • 현실적으로 import, export를 그대로 사용하기에 어려운 현실적인 이유 2가지
      1. 브라우저 호환성 이슈
      • 모듈을 지원하지 않는 브라우저가 있고, 버전에 따라 지원하지 않는 경우도 있다.
      1. 최신 문법
      • import와 export만이 최신 문법이 아니다.
      • 최신 문법 측면에서 브라우저 호환성의 문제는 더 커진다.

# 번들러

# 번들러

  • 시작되는 Javascript 파일 하나를 지정하면, 그 파일 안에서 다른 Javascript을 불러서 실행할 경우, 브라우저에서 실제로 Javascript 파일을 불들이기 전에 미리 그 많은 파일을 하나의 파일로 만들어 주는 것.
    • HTML 내에서는 만들어진 하나의 파일만 스크립트 태그에 연결해 두면, 위의 문제가 해결된다.
    • 즉, 번들링 과정을 통해 모듈을 부르는 스펙을 못쓰던 상황에서 다른 자바스크립트 파일을 불러 쓸 수 있게 되었다.

# Webpack

  • 번들러 중 하나
    • Javascript 파일이 아닌 파일들도 번들링 과정을 통해 아웃풋으로 낸다.
    • 여러 개의 Javascript 파일을 하나로 합치는 소프트웨어를 만들고 보니, 더 다른일도 할 수 있지 않을까 해서 더 유용한 기능도 추가 되었다.
      • 필요없는 소스코드를 제거(주석 등), 이미지 파일의 크기가 크면 압축해서 줄이는 작업, 어글리파이(중요한 코드를 알아보기 힘들게 만드는 작업), 트랜스파일링 등등
    • 트랜스파일링 : 인간이 이해할 수 있는 프로그래밍 언어를 컴퓨터가 이해할 수 있는 프로그래밍 언어로 변환.

# Babel

  • 트랜스파일러 중 하나.
    • 최신 문법의 Javascript를 과거 문법의 Javascript로 변환하면 원하는 문법으로 개발을 하고, 안정적인 버전으로 서비스를 할 수 있음
    • 번들링 과정에 트랜스 파일러를 끼워넣어서 최신 버전의 문법을 하위 버전으로 문법으로 바꾼다.
    • Typescript도 Typescript가 제공하는 트랜스파일러를 이용해서 원하는 javascript 버전으로 변경
